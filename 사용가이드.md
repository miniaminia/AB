## 🚀 5분 만에 시작하기

### 1단계: Firebase 설정 (처음 한 번만)

#### ① Firebase 프로젝트 만들기
1. https://console.firebase.google.com/ 접속
2. "프로젝트 추가" 클릭
3. 프로젝트 이름 입력 (예: `우리팀-ab테스트`)
4. Google 애널리틱스는 끄고 진행
5. "프로젝트 만들기" 클릭

#### ② 데이터베이스 만들기
1. 왼쪽 메뉴 "빌드" > "Realtime Database" 클릭
2. "데이터베이스 만들기" 클릭
3. 위치: `asia-southeast1` (싱가포르) 선택
4. 보안 규칙: "테스트 모드에서 시작" 선택
5. "사용 설정" 클릭

#### ③ 웹 앱 등록
1. Firebase 홈 화면에서 웹 아이콘 `</>` 클릭
2. 앱 닉네임 입력 (예: `AB테스트`)
3. "앱 등록" 클릭
4. 설정 정보 복사 (중요!)

```javascript
// 이런 형태로 나옵니다
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "your-project.firebaseapp.com",
  databaseURL: "https://your-project-default-rtdb...",
  // ... 등등
};
```

#### ④ HTML 파일에 설정 붙여넣기
1. `ab-test-firebase.html` 파일 열기
2. 71번째 줄 근처 `firebaseConfig` 찾기
3. `YOUR_API_KEY` 등을 실제 값으로 교체

### 2단계: 배포하기

#### 방법 A: GitHub Pages (추천)
```
1. GitHub에 로그인
2. 새 저장소 만들기 (public)
3. ab-test-firebase.html 파일 업로드
4. Settings > Pages
5. Source: main branch 선택
6. Save 클릭
7. 나오는 URL 복사 (예: https://username.github.io/repo/ab-test-firebase.html)
```

#### 방법 B: 로컬에서 테스트
```
1. 파일을 브라우저로 그냥 열기
2. Chrome에서 파일 열기 권장
```

### 3단계: 사용하기

#### 선생님/팀장이 할 일
```
1. 사이트 접속
2. 팀 이름 입력 (예: "A팀-1월12일")
3. 비밀번호 입력 (예: "1234")
4. 화면 A, B에 각각 이미지 업로드
5. "공유 링크 복사" 클릭
6. 학생들/팀원들에게 링크 전달
```

#### 학생들/팀원들이 할 일
```
1. 받은 링크로 접속
2. 비밀번호 입력 (선생님이 알려준 것)
3. 두 화면 비교
4. 마음에 드는 쪽 투표 버튼 클릭
5. 왜 선택했는지 이유 작성 (선택사항)
6. "투표하기" 클릭
```

#### 결과 확인
```
모든 사람이 실시간으로 같은 화면을 봅니다!
- 현재 투표 수
- 퍼센티지
- 다른 사람들이 남긴 선택 이유
```

## 💡 실전 활용 예시

### 예시 1: 디자인 수업에서
```
상황: 학생 5팀이 각자 앱 디자인 2가지 버전을 만들었습니다.

1. 각 팀별로 테스트 생성
   - 팀 이름: "1팀-회원가입화면", "2팀-회원가입화면" ...
   - 비밀번호: 모든 팀 같은 비밀번호 사용 (예: "design2024")

2. 각 팀이 자기 화면 업로드
   - A버전, B버전 스크린샷 업로드

3. 전체 학생들에게 링크 공유
   - 1팀부터 5팀까지 모든 링크 공유
   - 학생들이 각 팀 화면 보고 투표

4. 결과 분석
   - 어느 버전이 더 인기 있는지
   - 왜 그런지 이유 확인
   - 다음 수업 시간에 피드백
```

### 예시 2: 회사 디자인팀에서
```
상황: 새 기능 디자인 2가지 시안 중 선택

1. 디자이너가 테스트 생성
   - 팀 이름: "신기능-결제화면"
   - 비밀번호: "company123"

2. 두 시안 업로드

3. 전사 직원들에게 공유
   - 슬랙/이메일로 링크 전달

4. 투표 결과로 최종 선택
   - 실제 사용자 의견 반영
   - 구체적인 피드백 수집
```

## 🔒 보안 관련 Q&A

**Q: 누가 내 데이터를 볼 수 있나요?**
A: 링크를 받고 비밀번호를 아는 사람만 볼 수 있습니다.

**Q: 비밀번호는 안전한가요?**
A: SHA-256 암호화로 저장되어 원본 비밀번호는 아무도 볼 수 없습니다.

**Q: 학생이 여러 번 투표할 수 있나요?**
A: 아니요, 한 번만 투표 가능합니다. (브라우저 기준)

**Q: 다른 팀 데이터를 삭제할 수 있나요?**
A: 비밀번호를 모르면 불가능합니다.

## 🐛 자주 발생하는 문제

### "Permission denied" 에러
```
원인: Firebase 설정 문제
해결: 
1. Firebase 콘솔 > Realtime Database > 규칙
2. 규칙 확인 (FIREBASE_SETUP.md 참고)
3. "게시" 버튼 클릭
```

### 이미지가 업로드 안 됨
```
원인: 파일 크기 또는 형식 문제
해결:
1. 이미지 크기 2MB 이하로 줄이기
2. JPG 또는 PNG 형식 확인
3. 스크린샷을 다시 찍어보기
```

### 실시간 업데이트가 안 됨
```
원인: Firebase 설정 정보 오류
해결:
1. HTML 파일에서 firebaseConfig 다시 확인
2. databaseURL이 정확한지 확인
3. 브라우저 콘솔(F12)에서 에러 확인
```

### 비밀번호를 잊어버렸어요
```
해결:
1. Firebase 콘솔 접속
2. Realtime Database > 데이터 탭
3. 해당 팀 데이터 직접 삭제
4. 새로 만들기
```

## 🎓 수업 활용 팁

### 준비물
- [ ] 프로젝터 (결과 화면 띄우기용)
- [ ] 학생들 스마트폰/노트북
- [ ] WiFi 연결

### 진행 순서
1. **설명 (5분)**
   - AB테스트가 뭔지 설명
   - 오늘 할 활동 소개

2. **업로드 (10분)**
   - 각 팀이 자기 작업물 업로드
   - 링크 수집

3. **투표 (15분)**
   - 학생들이 모든 팀 작업물 보고 투표
   - 이유 작성 독려

4. **결과 공유 (10분)**
   - 프로젝터로 결과 화면 띄우기
   - 각 팀별 피드백 정리
   - 좋은 점, 개선점 토론

### 평가 기준
- 투표 수 (30%)
- 피드백 내용 (40%)
- 디자인 완성도 (30%)
